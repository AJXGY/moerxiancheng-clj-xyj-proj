# Torch MVP Dashboard

`mvp_dashboard.py` adds a lightweight front-end/back-end wrapper around `torch_infer_mvp.py`.

## What it does

- serves a browser UI from `ui/`
- launches `torch_infer_mvp.py` runs in the background
- supports three execution modes:
  - local Python
  - `docker run --gpus all` with a specified image
- reads each run's `report.json` and renders:
  - estimate vs measured timing cards
  - op-family and region breakdowns
  - profiler top CUDA ops
  - top estimated contributors
  - captured stdout/stderr logs
  - graph-viz assets generated by `export_graph_viz.py`

## Start the dashboard

From the repo root:

```bash
python3 mvp_dashboard.py
```

Then open `http://127.0.0.1:8123`.

## Suggested default flow

- runner: `docker_run_image`
- image: `ubuntu2204-torch26-py:latest`
- model path: one of the locally downloaded models under `downloads/`
- warmup: `2`
- benchmark_repeat: `5`
- profile_repeat: `10`

Graph export is enabled by default. Each run can generate a `graph_viz/` subdirectory and the UI will preview the estimate graph or graph index directly.

This mode is the most robust because it starts a fresh GPU-enabled container for each run.
